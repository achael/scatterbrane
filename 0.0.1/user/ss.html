SimScat Module¶

Important Notes¶

This section highlights some important aspects of the module.

The first thing to do is load the SimScat class and logging module

from simscat import SimScat
import logging
logging.basicConfig(level=logging.INFO) # for example

The SimScat object holds the parameters for the scattering screen and functions for generating images. To initialize a simulation you contruct an instance of the screen and set parameters

import numpy as np
s = SimScat(np.zeros((64,64)),1,nphi=4096,screen_res=5,wavelength=3e-6,r_inner=100)

In this example I have set the source to be empty with a pixel size of 1 :math:’muas’. Upon initialization, SimScat creates a version of the source image whose pixel size is chosen to match the screen.

Warning

The resolution of the scattered (iss) and source images (isrc) may not match the original resolution element you provided SimScat. You should access the property through dx.

Construction of a SimScat instance will check some properties:

  • Screen covers the source image.
  • Source image is square.
  • Inner turbulence scale is larger than the phase coherence length.
  • Resolution element of the screen is larger than the inner turbulence scale.
  • (warning) Is the image resolution approaching the refractive scale.

We can now proceed to generate the phase screen and scatter the source image:

s.generateScreen()
s.scatter()

Class Methods¶

class simscat.SimScat(model, dx, nphi=4096, screen_res=2, wavelength=1.3e-06, dpc=8400, rpc=5800, r0='sgra', r_outer=10000000, r_inner=12, alpha='kolmogorov', anisotropy=2.045, pa=78, match_screen_res=False)¶

Scattering simulation object.

Parameters:
  • model – The input model or source image which may be a data cube for polarization (stokes axis=0).
  • dx – Resolution element of model in uas.
  • nphi – (optional) Number of pixels in a screen.
  • screen_res – (optional) Size of a screen pixel in units of \(R_0\)
  • wavelength – (optional) Observing wavelength in km.
  • dpc – (optional) Observer-Source distance in parsecs.
  • rpc – (optional) Observer-Scatterer distance in parsecs.
  • r0 – (optional) Phase coherence length along major axis as preset string or in km.
  • r_outer – (optional) Outer scale of turbulence in units of \(R_0\)
  • r_inner – (optional) Inner scale of turbulence in units of \(R_0\)
  • alpha – (optional) Preset string or float to set power-law index for tubulence scaling (e.g., Kolmogorov has \(\alpha= 5/3\))
  • anisotropy – (optional) Anisotropy for screen is the EW / NS elongation.
  • pa – (optional) Orientation of kernel’s major axis, E of N, in degrees.
Returns:

An instance of a scattering simulation.
Example:

s = ScatSim(m,dx,nphi=2**12,screen_res=5.,wavelength=3.0e-6,dpc=8.4e3,rpc=5.8e3)

where s is the class instance, m is the image array, nphi is the number of screen pixels, wavelength is the observing wavelength.

Note

\(R_0\) is the phase coherence length and Sgr A* defaults are from Bower et al. (2006).

chatter()¶

Print information about the current scattering simulation.

Note

Many parameters are cast as integers.

checkSanity()¶

Check that screen parameters are consistent.

generateScreen(seed=None, save_phi=None)¶

Generate screen phases

Parameters:
  • seed – Seed for random number generator
  • save_phi – To save the screen, set this to the filename.
load_src(stokes=(0, ))¶

Load the source image from model (changes size and resolution to match the screen).

Parameters:stokes – Stokes parameters to consider.
readScreen(filename)¶

Read in screen phases from a file.

Parameters:filename – File containing the screen phases.
saveSettings(filename)¶

Save screen settings to a file.

Parameters:filename – settings filename
scatter(sp=0, stokes=(0, ))¶

Generate the scattered image.

Parameters:
  • sp – Number of screen pixels to roll the screen along in the x-dimension.
  • stokes – Stokes parameters to include in the scattering calculation.

Note

The screen stays fixed after each roll so the effect of sp will be cumulative.

simscat.utilities.regrid(a, inx, idx, onx, odx)¶

Regrids array with a new resolution and pixel number.

Parameters:
  • a – input square, 2D array
  • inx – number of input pixels on a side
  • idx – input resolution element
  • onx – number of output pixels on a side
  • odx – output resolution element
Returns:

Array regridded to the new resolution and field of view.
